NAME

mctally — request list of installed packages matching a regular expression

SYNOPSIS

mctally [-h host(s)] [-a] [-l] [-p port] [-t] [-u] [-v] [packages_regex]

DESCRIPTION

mctally requests installed package information from one or more hosts via mctallyd(8). It can also request load averages, active logins, uptime and uname information.

The following command line options are available:

-h host(s)
Specify the mctallyd(8) host(s) to be queried. If not specified, localhost is queried.
-a
Request load averages.
-l
Request active logins.
-p port
Specify the TCP port where mctallyd(8) is listening.
-t
Request uptime.
-u
Request uname information.
-v
Display the version of mctally.
packages_regex
Request a list of installed packages whose name matches packages_regex. packages_regex is an ECMAScript regular expression.

FILES

mctally uses libCredence for encryption and authentication. Hence it requires credence key files.

~/.credence/id_ed25519
The user's credence private key file, created with credence(1). This file should be owned by the user and have permissions 0600. It must contain the private part of an Ed25519 RSA key pair.
~/.credence/id_ed25519.pub
The user's credence public key file, created with credence(1). This file should be owned by the user and have permissions 0600. It must contain the public part of an Ed25519 key pair.
~/.credence/known_keys
The user's credence known services file. This file must contain the public keys of mctallyd(8) from the mctallyd(8) hosts being queried (one per line). It is used to authenticate instances of mctallyd(8).
