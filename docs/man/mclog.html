NAME

mclog — view log messages from multicast or files

SYNOPSIS

mclog [-c configFile] [-F filterExpression] [-d] [files...]

DESCRIPTION

mclog may be used to view log messages multicasted from instances of mclogd(8). It may also be used to view binary log files that were written by mclogd(8).

The following command line options are available:

-c configFile
Specify the path to the mclogd(8) configuration file. The default is /usr/local/etc/mclogd.cfg
-F filterExpression
Only display messages which match the given filterExpression.
-d
Enable debugging messages on stderr.
files...
If present, mclog will read messages from the given files. If not present, mclog will read messages that were multicasted by mclogd(8).

FILES

mclog uses libCredence for encryption and authentication. Hence it requires credence key files.

~/.credence/id_ed25519
The user's credence private key file, created with credence(1). This file should be owned by the user and have permissions 0600. It must contain the private part of an Ed25519 RSA key pair.
~/.credence/id_ed25519.pub
The user's credence public key file, created with credence(1). This file should be owned by the user and have permissions 0600. It must contain the public part of an Ed25519 key pair.
~/.credence/known_keys
The user's credence known services file. This file must contain the public keys of mclogd(8) from the mclogd(8) hosts being queried (one per line). It is used to authenticate instances of mclogd(8).
