NAME

mclogd.cfg — mclogd configuration file.

SYNOPSIS

mclogd.cfg

DESCRIPTION

mclogd.cfg is the configuration file for mclogd(8). Its default location is /usr/local/etc/mclogd.cfg and an example can be found in /usr/local/etc/mclogd.cfg.example

FILE FORMAT

mclogd.cfg contains multiple stanzas. Stanzas are opened with a name and { and closed with };. There are five valid top-level stanza names: service, loopback, filters, multicast and files.

Comments start with # and continue to the end of the line. Empty lines are ignored.

service stanza

The service stanza contains settings for authentication.

keyDirectory = path;
The credence key directory.

An example service stanza is shown below.

    service {
        keyDirectory = "/usr/local/etc/mclogd";
    };

loopback stanza

The loopback stanza is used to enable listening on the IPv4 loopback and/or IPV6 loopback, and on which UDP port to listen.

listenV4 = boolean;
Should mclogd(8) listen on the IPv4 loopback address? The default is true.
listenV6 = boolean;
Should mclogd(8) listen on the IPv6 loopback address? The default is false.
port = port;
The UDP port on which mclogd(8) should listen for local messages (via the loopback address). The default value is 3737.

An example loopback stanza is below.

    loopback {
       listenV4 = true;    # listen on 127.0.0.1?
       listenV6 = false;   # listen on ::1?
       port = 3737;        # port on which to listen (default 3737)
    };

filters stanza

The filters stanza may be used to define named filter expressions or filter expression fragments which can later be referenced by name (with a preceding dollar sign $). This makes the construction of longer filter expressions a bit more convenient. Note that a name must be fully defined set before referenced; referencing a name before its value is set is undefined.

<name> = "<filter expression>";
Creates a filter expression named <name> with value <filter expression>.

An example filters stanza is shown below. Note how we refer to previously defined names apps and myhosts when defining myapps and daemons and myhosts when defining mydaemons.

    filters {
        apps = "ident = /mc(block|curtain|rover|tally)|qmcrover/";
        daemons = "ident = /mc(block|curtain|log|rover|tally|weather)d/";
        myhosts = "host = /.+.(mcplex.net|rfdm.com)/";
        myapps = "($apps) && ($myhosts)";
        mydaemons = "($daemons) && ($myhosts)";
    };
See the FILTER EXPRESSIONS section for filter grammar.

multicast stanza

The multicast stanza is used to set the interface that mclogd(8) will use for multicast as well as the multicast group and UDP port that will be used.

For IPv4 multicast, two settings are needed: intfAddr and groupAddr.

intfAddr = "<IPv4 address>";
The IPv4 address of the interface on which mclogd(8) should send and receive IPv4 multicast messages.
groupAddr = "<IPv4 multicast address>";
The IPv4 multicast group address to which we will send log messages we received via the loopback and from which we will receive log messages from other instances of mclogd(8) via multicast. The default value is 239.108.111.103

For IPv6 multicast, three settings are needed: intfAddr6, groupAddr6 and intfName.

intfAddr6 = "<IPv6 address>";
The IPv6 address of the interface on which mclogd(8) should send and receive IPv6 multicast messages.
groupAddr6 = "<IPv6 multicast address>";
The IPv4 multicast group address to which we will send log messages we received via the loopback and from which we will receive log messages from other instances of mclogd(8) via multicast. The default is ff02::006d:636c:6f67
intfName = "<interface name>";
The name of the interface on which we will send and receive IPv6 multicast messages. This interface must have the address specified as intfAddr6.

Both IPv4 and IPv6 multicast utilize the same UDP port number and output filter expression.

port = <port number>;
The port number to use for sending and receiving multicast messages. the default is 3737.
outFilter = "<filter expression>";
A filter expression to select the log messages which mclogd(8) will send via multicast. The default value is empty, which will cause mclogd(8) to send all local messages (received on the loopback) to the IPv4 and IPv6 multicast groups. See the FILTER EXPRESSIONS section for filter grammar.

An example multicast stanza is shown below.

   multicast {
      #------------------------------------------------------------------
      #  For IPv4, we need the interface address and the multicast group
      #  address.
      #------------------------------------------------------------------
      intfAddr = "192.168.168.42";
      groupAddr = "239.108.111.103";
      #------------------------------------------------------------------
      #  For IPv6, we need the interface address, the multicast group
      #  address and the interface name.
      #------------------------------------------------------------------
      intfAddr6 = "fd60:3019:f4a:6aaf::39";
      groupAddr6 = "ff02::006d:636c:6f67";
      intfName = "en0";
      #------------------------------------------------------------------
      #  We use the same destination port for IPv4 and IPv6.
      #------------------------------------------------------------------
      port = 3737;

      outFilter = "$mydaemons";
   };

files stanza

An example files stanza is shown below.

   files {
      logDirectory = "/usr/local/var/logs";
      logs {
          { filter="$mydaemons"; path="%H/%I"; user="dwm"; group="staff";
            perms=0644; period=1d; size=500k; keep=7; compress="bzip2"; },
          { filter="$myapps"; path="%H/myapps"; perms=0600; }
      };
   };

FILTER EXPRESSIONS

Below is the pseudo-EBNF for the filter expression grammar.

    expression: "(" expression ")"
      | "!" expression
      | expression "||" expression
      | expression "&&" expression
      | "facility" numerical_op facility_value
      | "severity" numerical_op severity_value
      | "host" equality_op quoted_string
      | "host" equality_op regular_expression
      | "ident" equality_op quoted_string
      | "ident" equality_op regular_expression
      | "pid" numerical_op unsigned_int
      | "msg" equality_op quoted_string
      | "msg" equality_op regular_expression;

    numerical_op: "<" | "<=" | ">" | ">=" | equality_op;
    equality_op: "=" | "!=";
    quoted_string: "'" [^']* "'";
    regular_expression: "/" [^/]* "/";
    unsigned_int: [0-9]{1,10};

    facility_value: kernel | user | mail | daemon | auth | syslog
      | lpr | news | uucp | cron | authpriv | ftp | local0 | local1
      | local2 | local3 | local4 | local5 | local6 | local7;

    severity_value: debug | info | notice | warning | err | crit
      | alert | emerg;

FILTER EXPRESSION EXAMPLES

Select messages with a severity greater than debug from host foo.mcplex.net:

    (severity > debug) && (host = 'foo.mcplex.net')

Select messages with identifier mcroverd or mctallyd from hosts outside of the mcplex.net domain:

    (ident = /(mcroverd|mctallyd)/) && (host != /.*\.mcplex\.net/)

FILES

CONFIGURATION FILE

The standard location of the mclogd(8) configuration file is /usr/local/etc/mclogd.cfg

An example mclogd(8) configuration is included in /usr/local/etc/mclogd.cfg.example
